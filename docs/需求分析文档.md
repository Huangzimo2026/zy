\## 一、项目概述

\- \*\*项目名称\*\*：小航AI助手 - 郑航校园智能咨询系统

\- \*\*项目目标\*\*：为郑州航院大一新生、在校老生、教师三类用户提供校园信息智能问答服务。

\- \*\*技术栈\*\*：Streamlit + 硅基流动 API (Qwen/Qwen2.5-7B-Instruct) + Markdown 数据文件。



\## 二、用户分析

\### 2.1 大一新生（高优先级）

\- \*\*特点\*\*：对校园不熟悉，容易焦虑，容易受骗。

\- \*\*高频问题\*\*：报到流程、学费缴纳、宿舍环境、防骗指南。



\### 2.2 在校老生（中优先级）

\- \*\*特点\*\*：办事高效，追求简洁。

\- \*\*高频问题\*\*：在读证明、校园卡补办、转专业、图书馆时间。



\### 2.3 教师（中优先级）

\- \*\*特点\*\*：需要政策依据和精确联系人。

\- \*\*高频问题\*\*：差旅报销、调课申请、设备报修、科研申报。



\## 三、功能需求（4个P0必做）

| 编号 | 功能名称 | 描述 | 优先级 |

|------|---------|------|--------|

| P0-1 | 校园问答 | 用户输入问题，AI基于学校资料回答 | P0 |

| P0-2 | 身份选择 | 切换“新生/在校生/教师”三套Prompt | P0 |

| P0-3 | 推荐问题按钮 | 12个常见问题，一键填入输入框 | P0 |

| P0-4 | 电话黄页静态页 | 不依赖API，展示常用电话 | P0 |



\## 四、应用流程

用户打开小航 -> 选择身份 -> 点击推荐问题或手动输入 -> AI调用硅基流动API回答 -> 显示回答并标注来源。若API失败，可查看静态电话黄页兜底。



（此处可粘贴你的流程图，若无图可描述）



\## 五、数据设计

\- \*\*数据格式\*\*：Markdown（人写友好，AI直接读）。

\- \*\*文件清单\*\*：

&#x20; - `data/01\_新生入学.md`：报到、宿舍、学费

&#x20; - `data/02\_办事流程.md`：证明、补卡、转专业

&#x20; - `data/03\_电话黄页.md`：应急、行政电话

&#x20; - `data/04\_应急防骗.md`：防骗、心理援助



\## 六、Prompt设计

\- \*\*身份分流\*\*：新生（详细/鼓励）、在校生（简洁/效率）、教师（专业/政策）。

\- \*\*别名词典\*\*：覆盖“郑航/ZUA/航院”等7组别名。

\- \*\*6条防幻觉硬规则\*\*：禁止编造电话/金额，强制标注来源，心理危机兜底等。



\## 七、用户边界声明

\- \*\*能聊\*\*：新生报到、办事流程、电话黄页、应急防骗。

\- \*\*不能聊\*\*：查个人成绩/课表（不接入学校系统）。

\- \*\*数据更新日期\*\*：2026-07-17（以官方为准）。



\## 八、不做的事

\- 不做向量库/RAG（大一阶段不引入）。

\- 不做LangChain（仅用requests库）。

\- 不做用户登录（不存账号密码）。

\- 不做数据库（用Markdown代替）。

\- 不做部署上线（本地能跑即可）。
